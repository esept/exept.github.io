类和对象

类的定义

成员变量是直接定义在类当中的，在方法外面
成员方法不要谢static关键词

使用

通常情况下，一个累并不能直接使用，需要根据类创建一个对象，才能使用

  1. 导报，指出需要使用的类，在什么位置
    import 包名称
    import···

2.创建，格式：
类名称 对象名 = new 名称();

3.使用
分为两种情况
成员变量: 对象名.成员变量名
成员方法: 对象名.成员方法名

如果成员变量没有进行赋值,那么会有一个默认值

###适用对象类型作为方法的参数
当一个对象作为参数,传递到方法中时,实际上传递进去的是对象的地址值

使用对象类型作为方法的返回值

局部变量和成员变量

  1. 定义的位置不一样
    局部变量: 在方法的内部
    成员变量: 在方法的外部,直接写在类中

  2. 作用范围不一样
    局部变量: 只有方法当中可以使用,出了方法不可继续使用
    成员变量:整个类可以通用

  3. 默认值不一样
    局部变量: 没有默认值,如果想使用,必须手动赋值
    成员变量: 有默认值

  4. 内存的位置不一样
    局部变量: 位于栈内存
    成员变量: 位于堆内存

  5. 生命周期不一样
    局部变量: 随诊方法进栈而产生,随着方法出栈而消失
    成员变量: 随着对象创建而产生,随着对象被垃圾回收而消失

封装

体现有两点:
方法就是一种封装
关键字private也是一种封装(私有化)

封装就是将一些细节信息隐藏起来,对于外界不可见

exemple

定义person的年龄时,无法阻止不合理的数值被设置进来
解决: 使用private进行修饰,那么本类中仍然可以随意访问,但是超出本类范围之外就不能再直接访问
简介访问成员变量,就是间接定义一对get/set方法

this 关键字
如果需要访问本类当中的成员变量,需要使用格式:this.成员变量名
通过谁调用的方法,谁就是this

构造方法
构造方法是专门用来创建对象的方法,当我们通过关键字new来创建对象时,其实就是在调用构造方法.
格式:
public 类名称(参数类型 参数名称){
方法体
}
注意事项:
构造方法的名称必须和所在的类名称完全一样(大小写)
构造方法不要写返回值类型,练void也不写
构造方法不能return一个具体的返回值
如果没有编写任何构造方法,那么便一起将会默认赠送一个构造方法,没有参数、方法体什么事情都不做
一旦编写了一个构造方法,那么便一起将不再赠送
构造方法也是可以重载的

标准的类
通常有四个组成部分
·所有的成员变量都是private
·每一个成员变量都有getter/setter
·编写一个无参数的构造方法
·编写一个全参数的构造方法
